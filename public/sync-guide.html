Sync Offline para OPENAUDITOR

El HTML autónomo genera un único payload JSON y puede enviarlo por `fetch` al destino que configures. Si falla, siempre puede exportar el JSON manualmente.

Apps Script / Google Sheets

La forma gratis más simple de empezar. Recibe el JSON, crea una hoja por plantilla y amplía columnas automáticamente según las preguntas de esa plantilla.

  1. Creá un nuevo proyecto de Apps Script.
  2. Pegá el ejemplo de Apps Script.
  3. Publicalo como Web App con acceso para quien tenga el enlace.
  4. Copiá la URL final en `Configuración -> Sync offline y destinos`.

Cada plantilla escribe en su propia pestaña, por ejemplo `TPL - Checklist de seguridad`. Si agregás preguntas nuevas a esa plantilla, el script suma columnas nuevas sin mezclar datos con otras plantillas.

PocketBase

La opción open source recomendada para crecer sin cambiar de flujo. El ejemplo recibe el JSON, crea un registro y puede guardarse como middleware o endpoint propio delante de PocketBase.

  1. Desplegá PocketBase en una VPS o equipo propio.
  2. Creá una colección `inspections_imports` con campos para `templateId`, `templateName`, `completedAt` y `payload`.
  3. Usá el ejemplo de PocketBase como receptor HTTP.
  4. Configurá el destino en OPENAUDITOR con el tipo `PocketBase`.

Supabase

Conviene usar una Edge Function como puente. El payload es el mismo que ves arriba; podés persistirlo en Postgres y mover fotos a Storage.

Webhook Genérico

Si ya tenés n8n, Make, Zapier, un ERP o tu propia API, usá el tipo `Webhook JSON` y enviá el payload tal cual.

GitHub / Gist

Sirve como respaldo liviano o bitácora externa usando la API de GitHub. Requiere token y no es la mejor opción para producción o muchas fotos.

OPENAUDITOR Remoto

Podés usar otra instancia como receptor apuntando al endpoint `/api/import/standalone` de esa instalación.
